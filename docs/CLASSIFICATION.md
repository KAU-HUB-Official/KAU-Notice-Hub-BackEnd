# 공지 분류 기준

## 범위
이 문서는 FastAPI 백엔드가 공지의 대상자 대분류, 중분류, 세부 홈페이지 필터를 계산하는 기준을 정의한다.

구현 위치:

```text
app/classification.py
```

분류값은 저장된 수동 값이 아니라 요청 시점에 결정적 함수로 계산한다.

## 대상자 대분류

노출 순서:

1. `전 구성원 공통`
2. `학부 재학생(학과/전공별)`
3. `신입생·저학년`
4. `재학생 비교과·글로벌 프로그램`
5. `취업·창업 준비생`
6. `대학원생`
7. `평생·전문교육원`
8. `그 외`

## 세부 홈페이지 필터 허용 대분류

`source` 쿼리 필터는 아래 대분류에서만 적용한다.

- `학부 재학생(학과/전공별)`
- `대학원생`
- `평생·전문교육원`

다른 대분류에서는 `source` 쿼리가 들어와도 에러를 내지 않고 무시한다.

## 중분류

| 대분류 | 중분류 |
| --- | --- |
| `전 구성원 공통` | `일반`, `학사`, `장학/대출`, `입찰`, `행사` |
| `학부 재학생(학과/전공별)` | `공과대`, `AI융합대`, `항공경영대`, `그 외 학부` |
| `신입생·저학년` | `새내기성공센터`, `드림칼리지디자인` |
| `재학생 비교과·글로벌 프로그램` | `국제교류`, `첨단분야 부트캠프`, `산학협력`, `교수학습센터` |
| `취업·창업 준비생` | `대학일자리플러스센터`, `학과 취업공지` |
| `대학원생` | 없음 |
| `평생·전문교육원` | 없음 |
| `그 외` | `생활관`, `인권센터`, `학술정보관`, `LMS`, `입학처`, `박물관` |

중분류가 없는 대분류에서는 `group` 또는 `sourceGroup` 쿼리를 무시한다.

## 공식 홈페이지 중분류

공식 홈페이지 공지는 `category`와 `title`을 함께 보고 중분류를 계산한다.

| 키워드 | 중분류 |
| --- | --- |
| `입찰` | `입찰` |
| `학사`, `수강`, `졸업`, `성적`, `등록` | `학사` |
| `장학`, `대출`, `국가근로` | `장학/대출` |
| `행사`, `특강`, `설명회`, `세미나` | `행사` |
| 그 외 | `일반` |

## 복수 source 처리

원천 JSON의 `source_name`은 문자열 또는 배열일 수 있다.

정규화 결과:

- `source`: 첫 번째 source
- `sources`: 전체 source 배열

복수 source 공지는 모든 관련 source facet에 노출된다. 특정 대분류/중분류의 source facet을 계산할 때는 source를 하나씩 분리해 현재 필터 범위에 맞는 source만 노출한다.
